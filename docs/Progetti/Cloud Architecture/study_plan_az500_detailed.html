Architettura Sicurezza Azure (Zero-Trust) - Kitchen Manager

Ambito: Integrazione Identity & Security su Stack Ibrido

Il presente report descrive le strategie di mitigazione delle minacce e le configurazioni di hardening implementate nell'ambito dell'ammodernamento del progetto Kitchen Manager (React/Django) sfruttando la suite di sicurezza offerta da Microsoft Azure, con particolare orientamento a metodologie e compliance in perimetro AZ-500.

1. Identity & Access Management (IAM)

  • L'infrastruttura identitaria si basa largamente sulle direttive di Microsoft Entra ID per fornire un'autenticazione centralizzata. L'impiego pervasivo del Role-Based Access Control (RBAC) assicura la distribuzione mirata e misurabile dei privilegi operativi per i container logici in tutto lo stack del tenant.
  • Mediante una severa configurazione dei regolamenti di Conditional Access, i processi strategici impongono l'attivazione automatica della Multi-Factor Authentication (MFA) ed il confinamento rigido in caso di traffico logico proveniente al di fuori da determinati intervalli sicuri, respingendo efficacemente tentativi ostili mirati alle dashboard amministrative.

2. Isolamento di Rete (Datastore Protection)

  • Con l'obiettivo di contenere potenziali vulnerabilità perimetrali esposte al network pubblico, il data tier (PostgreSQL su Flexible Server) sfrutta ampie configurazioni di sicurezza isolanti tramite VNet nativa di Azure.
  • Tale disaccoppiamento esclude radicalmente interrogazioni di natura Internet, definendo il livello di interazione basato solo su Network Security Groups (NSG) rigorosi, imponendo al database di processare il traffico proveniente escusivamente dai dispositivi backend pre-identificati.

3. Sicurezza e Protezione Segreti Applicativi (Key Vault)

  • La presenza essenziale di secret strings, come stringhe transazionali del Database e API Key primarie per interpellare i linguaggi model-based esterni (l'AI di Google Gemini), impongono un design immune da injection nei vari .env file sui webservers di produzione.
  • L'impalcatura software subordina la sua esecuzione interrogando Azure Key Vault in autonomia. Le app ereditano il privilegio delegato via Managed Identity, instaurando meccaniche di fetch per decriptare i secrets critici "on-the-fly". Questa struttura preserva inviolata la segregazione tra gli amministratori di chiave infrastrutturali e i developer che materialmente rilasciano il software.

4. Governance e Audit Trailing

  • Qualunque operazione critica interfacciata con il management dei segreti o i profili autorizzativi produce log completi pronti per attività d'ingaggio difensive o revisioni e investigazione di accessi di natura forense.
