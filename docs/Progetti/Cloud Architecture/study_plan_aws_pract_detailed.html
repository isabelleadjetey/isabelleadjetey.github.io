AWS Certified Cloud Practitioner - Piano di Battaglia Dettagliato

(Periodo: 7 Aprile - 25 Aprile)

L'obiettivo di questo esame base è dimostrare di padroneggiare concetti generali ad alto livello. Questo piano mappa la teoria direttamente sull'architettura reale del tuo progetto Kitchen Manager (React + Vite + Django), offrendoti così la possibilità di applicare concetti enterprise alla tua codebase vera.

7 - 12 Aprile: La Teoria Mappata sul Kitchen Manager

Finisci il corso Essentials su AWS Skill Builder tenendo sempre a mente come i servizi ospiteranno il tuo codice.

Il Tuo Frontend (React + Vite):

  • Al momento generi i file statici usando vite build. Su AWS, l'intera cartella dist/ verrà caricata su Amazon S3, configurato come servizio di Static Website Hosting.
  • Per garantire tempi di risposta fulminei all'interfaccia usata dai camerieri e cuochi, S3 verrà collegato ad Amazon CloudFront, la CDN nativa che distribuirà il sito tramite le edge location in tutto il mondo garantendone anche la crittografia HTTPS e la protezione tramite WAF (Web Application Firewall).

Il Tuo Backend (Python / Django):

  • Il tuo codice back-end (in cui usi librerie come asgiref e anyio) richiede potenza di calcolo (Compute). Il modo più semplice e gestito per testare l'infrastruttura web su AWS è usare AWS Elastic Beanstalk (categoria PaaS).
  • Beanstalk configurerà tutto da solo: avvierà le istanze Amazon EC2 e aggiungerà un server di Auto Scaling. Se il ristorante si riempie il sabato sera, Beanstalk scalerà le risorse EC2 automaticamente senza il tuo intervento e ne ridurrà il numero il lunedì mattina (Elasticity).

Il Tuo Database:

  • I dati degli ordini staranno su Amazon RDS (Relational Database Service) con engine PostgreSQL.
  • RDS implementa i backup automatizzati e l'amministrazione delle patch. Sulla Virtual Private Cloud (VPC), collegherai Backend e RDS protetti l'uno dall'altro usando i Security Groups (permettono al backend l'accesso solo alla porta 5432).

Intelligenza Artificiale (AI) e API Keys:

  • Poiché hai curato tu stessa l'integrazione AI, il servizio di riferimento sarà Amazon Comprehend: il servizio di elaborazione del linguaggio naturale (NLP) perfetto per analizzare automaticamente sentiment e testi (potenzialmente di recensioni ristorante) senza addestrare modelli custom.
  • Gestione delle API Key: All'esame la best-practice aziendale impone di NON salvare MAI le chiavi per modelli AI o database nel codice. Il servizio da studiare per ruotare e gestire le chiavi è AWS Secrets Manager (o AWS Systems Manager Parameter Store).

Il Disaccoppiamento (Decoupling)

  • Un sistema integrato come l'AI bloccherebbe Django se eseguito sincronicamente col cliente in attesa. Il backend avvia il processo spedendo un file in una coda Amazon SQS.
  • Una AWS Lambda (serverless function) prenderà il lavoro e interrogherà le API dell'AI in background, usando le API Key tratte segretamente dal Secrets Manager.

13 - 18 Aprile: Le Prove del Fuoco

Fai simulazioni ufficiali. Concentrati sempre sul leggere i distrattori. Se una domanda parla di un database per documenti JSON (tipo MongoDB), ricordati che la risposta AWS è DynamoDB, non RDS! Ogni opzione sbagliata è in realtà la risposta giusta per un'altra domanda potenziale.

19 - 22 Aprile: Le Argomentazioni Normative (30% dei Punti)

  • Well-Architected Framework: I pilastri su cui si baserà il tuo Kitchen Manager.
  • Modello di Responsabilità Condivisa: AWS garantisce la sicurezza "del" cloud. Tu garantisci la sicurezza "nel" cloud.
  • Identity and Access Management (IAM): Implementerai in AWS le policy Least Privilege come faresti con i ruoli interni all'app per "Admin", "Cameriere" e "Cuoco".

23 - 25 Aprile: Vittoria

Rileggi i tuoi appunti prestando attenzione alle differenze nomenclature chiave (es: CloudFormation vs Beanstalk, CloudTrail per gli audit vs CloudWatch per le performance).

Il 25 Aprile, affronta l'esame con molta consapevolezza.
